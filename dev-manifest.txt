1.Писать код по текущим правилам ESLint:
- соблюдать отступы, ставить точки с запятыми ; где это требуется
- проверять, что импорты через одинарные кавычки
- проверять, что jsx атрибуты через двойные кавычки
- не заглушать правила в коде через слэши // и /* */

2. Устанавливать пакеты только через yarn:
Пояснение: при установке через npm появляется package-lock.json, который не рекомендуется иметь вместе с yarn.lock

3. Стараться не смешивать стили с логикой в .js/.jsx файлах. Выносить стили в отдельный .css файл или модуль

4. Задавать хорошие наименования компонентов/переменных, учитывая как они называются в фигме

5. Основная работа ведется в своих ветках.
- изменения из своей ветки заливаем в общую dev-ветку (не main). Совместное ревью
- в main пушим готовую протестированную фичу и только после совместного ревью
